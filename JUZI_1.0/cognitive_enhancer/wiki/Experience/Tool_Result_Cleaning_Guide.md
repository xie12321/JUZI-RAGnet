# 工具结果清洗指南

当 web_search 等工具返回的内容包含大量 HTML、广告、导航文字时，请执行以下步骤：

## 步骤1：删除所有无意义行
删除包含以下关键词的行：
- `[![](`、`<img`、`Image`、`![Image`
- `首页`、`预报服务`、`预警服务`、`防灾减灾`、`科普服务`
- `Thank you for your patience`、`loading`、`spinner`
- `WinterCast`、`{stormName}`、`%`（百分比符号单独出现时）
- 行首为 `*` 或 `-` 且内容不包含数字、温度符号的列表项

## 步骤2：定位有效数据块
保留包含以下模式的行：
- 数字 + `°C` 或 `℃` 或 `温度`
- 汉字天气词：`多云`、`晴`、`阵雨`、`雷阵雨`、`阴`、`雨`
- 风力风向：`风`、`级`、`东南`、`西北`

## 步骤3：提取关键字段
从保留的行中，尽量提取以下字段：
- `temperature`: 类似 `23℃~30℃` 或 `最高31°C`
- `condition`: `多云`、`晴` 等
- `wind`: `东南风3-4级`
- `humidity`: 如果有 `湿度 65%` 则保留

## 步骤4：输出 JSON
输出格式为：
```json
{"temperature": "...", "condition": "...", "wind": "...", "humidity": "..."}
```
如果某字段缺失，值设为空字符串 ""。

## 示例
#### 原始输入：

### 1. 深圳 - 百度
```text
生活气象指数 晚上好，今日天气晴好，心情也要保持晴朗哦. 穿衣: 衬衫类. 23℃~30℃. 多云, 空气优.
```
#### 输出：

```json
{"temperature": "23℃~30℃", "condition": "多云", "wind": "", "humidity": ""}
```
